Sales Summary
Roles: Sales Rep · Director · Executive — 3 users per role, one login per run
10 steps
⚙
Setup — Admin login & user impersonation
Login once as admin → impersonate each of the 3 role users → navigate to Sales Summary
Auth Login Impersonation
▸
  • Navigate to /admin/signin and log in with ADMIN_EMAIL / ADMIN_PASSWORD.
  • For each user in the role group (e.g. David Farris, Michelle Hupfer, Rich Closer for Sales Rep): search by first name, select exact row by full name + company, confirm impersonation.
  • After impersonation, click "Sales Summary" in the left sidebar and wait for the page heading to be visible.
  • After all steps pass, exit impersonation before testing the next user.
⚠ Login happens only once per run. All 3 users are looped inside the same Playwright test.
3
Notification button — hover, open, tabs, close, outside-click
Verify the bell button opens a panel with Overdue / Upcoming / Activity tabs, and closes correctly
UI BehaviorAria
▸
  • Locate the notification bell via getByRole('button', { name: 'Notifications' }).
  • Hover over the button — a rectangular highlight should appear around it.
  • Click the bell → the notification panel opens (detected via [role="tablist"] containing an "Overdue" tab).
  • Click the Overdue tab — tab becomes active, content loads.
  • Click the Upcoming tab — tab becomes active, content loads.
  • Click the Activity tab — tab becomes active, content loads.
  • Click the bell again → panel closes (tabList.isVisible() returns false).
  • Click the bell once more → panel reopens.
  • Click the Sales Summary h2 heading (outside the panel) → panel closes.
⚠ Selector confirmed from live DOM: aria-label="Notifications", aria-pressed="false/true"
4a
Date filter — default shows yesterday & hover effects
Verify the date button defaults to yesterday's date, open the dropdown, hover each preset
Filter
▸
  • Read the date filter button text — it must contain yesterday's date in MMM D, YYYY format.
  • Click the date filter button to open the dropdown; wait for "Yesterday" to be visible.
  • Hover each preset option: Yesterday · Month-to-date · Previous month · Quarter-to-date · Previous quarter · Year-to-date · Previous year · Custom range.
  • Each hover should produce a visual highlight effect.
  • Press Escape to close the dropdown without changing the selection.
4b
Date filter — click each preset, verify date text updates
Click every preset option and confirm the date filter button label reflects the new range
Filter
▸
  • For each preset (Yesterday, Month-to-date, Previous month, Quarter-to-date, Previous quarter, Year-to-date, Previous year): open dropdown → click preset → wait 800ms for refresh → read date button text → verify it is non-empty (length > 4 chars).
  • Custom range: open dropdown → click Custom range → verify a calendar picker opens (detected by [role="gridcell"]) → press Escape.
4c
Date filter — KPI direction logic & chart visibility
For each multi-day preset: verify KPI numbers, percentage direction, color coding, and chart renders
KPIFilter
▸
  • Presets tested: Month-to-date · Previous month · Quarter-to-date · Previous quarter · Year-to-date · Previous year.
  • For each: select preset → read Contracted Sales, Units, Orders KPI card text → parse current value, previous value, and displayed % change.
  • Verify % direction: if current > previous → % must be positive (green). If current < previous → % must be negative (red).
  • Verify % magnitude is within 3% of the calculated value.
  • Verify the chart (canvas / recharts) is visible.
  • Verify the Orders nav bar shows a count.
4d
Date filter — custom range: multi-month & single-day
Test wide range selection, single-day selection (donut chart), and outside-click closes the picker
FilterKPI
▸
  • Wide range: open Custom range → click first calendar cell → click last calendar cell → click Apply → verify KPI cards update.
  • Single day: open Custom range → click the middle calendar cell twice (start = end) → click Apply → verify a donut chart appears (single-day view).
  • Outside click: open Custom range → click the Sales Summary heading → verify calendar picker disappears.
4e
Orders count — KPI box = Orders − Returns logic
Verify the Orders KPI equals the Orders filter count minus the Returns filter count
KPI
▸
  • Read the Orders KPI card value (e.g. 42).
  • Click the Orders nav tab to open the order type filter.
  • Select "Orders" → record the count shown in the nav bar.
  • Select "Returns" → record the count.
  • Verify: Orders count − Returns count = Orders KPI value.
  • Reset filter to "All" at the end.
⚠ Order type filter selector may need DOM discovery — currently detected via text matching "Order type / All orders / All types".
5a
Product filter — sections, selections, count, "Just this"
Verify Comp Plan and Other Sold Products sections exist, selection updates the label, "Just this" works
Filter
▸
  • Read the product filter button default label — should contain "Comp Plan (N)".
  • Open the dropdown — verify both "Comp Plan" and "Other sold products" sections are visible.
  • Select "Other sold products" only → Apply → verify filter label updates.
  • Select "Comp Plan" only → Apply → verify filter label updates.
  • Hover the first product item → "Just this" button appears → click it → Apply → verify label updates.
  • Check 3 checkboxes → Apply → verify count is reflected in the filter label.
5b
Product filter — NDC toggle & search by name
Toggle the NDC view on/off and search for 8 known product names using both short and full name
Filter
▸
  • Open product filter → find the NDC toggle (role="switch").
  • Turn NDC ON → verify row count increases (child NDCs visible).
  • Turn NDC OFF → verify row count decreases back to parent-only view.
  • For each of 8 products: type short prefix → verify at least 1 match; type full name → verify at least 1 match; clear input.
  • Products: ephedrine, erythromycin, fluorescein, methylene, papaverine, phenobarbital, procainamide, tacrolimus.
6
Customers filter — All / Current / Returning / New
Select each customer segment and confirm data loads for each option
FilterKPI
▸
  • For each option (All, Current, Returning, New): click "All customers" button → select option → Apply if shown → wait 800ms → verify "Contracted sales" KPI is visible (data loaded).
  • Record the filter label after each selection.
  • After "New", reset back to "All".
